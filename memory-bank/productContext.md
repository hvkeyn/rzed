# Product Context: Zed Code Editor

## Why Zed Exists

Zed создан чтобы решить фундаментальные проблемы существующих редакторов кода:

1. **Производительность**: Большинство редакторов (особенно Electron-based) страдают от лагов и высокого потребления ресурсов. Zed использует нативный рендеринг через GPU для мгновенной отзывчивости.
2. **Коллаборация из коробки**: В отличие от редакторов, где совместная работа — дополнение, в Zed это встроенная функция первого класса.
3. **AI-интеграция без посредников**: Прямая интеграция с LLM-провайдерами без необходимости в сторонних плагинах.
4. **Современный стек**: Использование Tree-sitter для парсинга, Rust для максимальной производительности и безопасности.

## Problems Solved

- **Задержка ввода**: Нативный рендеринг даёт мгновенный отклик на каждое нажатие.
- **Фрагментация инструментов**: Всё в одном — редактор, AI-ассистент, терминал, дебаггер, коллаборация.
- **Сложность настройки**: Работает "из коробки" с разумными дефолтами.
- **Платформенная несогласованность**: Единый код на Rust для всех платформ.

## How It Works

### Пользовательский опыт
- Открываешь папку — начинаешь работать (без индексации проекта)
- Command Palette (`Cmd+Shift+P`) для всех действий
- Agent Panel (`Cmd+Shift+A`) для AI-взаимодействия
- Встроенный терминал, дебаггер, поиск по проекту
- Vim mode для пользователей Vim
- Коллаборация через каналы и шаринг проектов

### Модель расширений
- Расширения пишутся на Rust (компилируются в WASM)
- Поддержка языков через Tree-sitter грамматики
- LSP-серверы, formatter'ы, темы
- WASI Preview 2 для sandbox-безопасности

### AI/Agent экосистема
- Agent Panel: чат с LLM, выполнение команд, работа с контекстом
- Inline Assistant: генерация/трансформация кода прямо в редакторе
- Context Servers (MCP): интеграция с внешними инструментами
- `/rules` — инструкции для AI-агентов
- Edit Prediction: предиктивное редактирование
- Множество LLM-провайдеров

## User Experience Goals

- **Мгновенная отзывчивость**: Каждое взаимодействие должно давать мгновенную обратную связь
- **Минималистичный интерфейс**: Без лишних элементов, только нужное
- **Прогрессивное раскрытие**: Простые вещи просты, сложные — доступны когда нужны
- **Консистентность**: Единый дизайн-язык на всех платформах
- **Доступность**: Клавиатурная навигация, screen reader support (через AccessKit)
